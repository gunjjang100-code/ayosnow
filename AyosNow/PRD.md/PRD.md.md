# PuntaGo PRD

## 1. 문서 목적

이 문서는 PuntaGo가 어떤 문제를 해결하는 서비스인지, 어떤 사용자가 어떤 기능을 써야 하는지 정리한 제품 요구사항 문서입니다.

쉽게 말해 PRD는 "무엇을 만들 것인가"를 정리하는 문서입니다. 개발자가 기능을 만들 때, 운영자가 서비스를 준비할 때, QA가 검수할 때 기준표처럼 사용합니다.

## 2. 제품 개요

### 제품명

PuntaGo

### 제품 유형

필리핀 홈서비스 마켓플레이스

### 한 줄 설명

PuntaGo는 필리핀 고객이 에어컨 청소, 배관, 전기, 청소, 이사 같은 홈서비스를 쉽고 빠르게 요청하고, 검증된 전문가와 연결될 수 있게 돕는 웹서비스입니다.

### 핵심 가치

- 고객은 여러 전문가의 견적을 비교하고 예약할 수 있습니다.
- 전문가는 고객 요청에 견적을 보내고 채팅으로 상담할 수 있습니다.
- 관리자는 카테고리, 지갑, 충전, 추천 보상, 운영 설정을 관리할 수 있습니다.
- 모바일에서도 고객/전문가/관리자 핵심 흐름을 사용할 수 있습니다.

## 3. 배경 및 문제 정의

필리핀 홈서비스 시장에서는 고객과 전문가가 서로를 믿고 빠르게 연결되기 어렵습니다.

고객이 겪는 문제:

- 어떤 전문가가 믿을 만한지 판단하기 어렵습니다.
- 전화나 메신저로 여러 명에게 따로 문의해야 해서 시간이 오래 걸립니다.
- 서비스 가격과 가능 날짜를 비교하기 어렵습니다.
- 긴급 서비스가 필요할 때 빠른 연결이 어렵습니다.

전문가가 겪는 문제:

- 안정적으로 고객 요청을 받기 어렵습니다.
- 견적, 채팅, 예약, 정산이 여러 도구에 흩어질 수 있습니다.
- 자신의 인증서, 작업 사진, 후기 같은 신뢰 자료를 보여주기 어렵습니다.

관리자가 겪는 문제:

- 전문가 승인, 카테고리, 충전, 지갑, 신고, 추천 보상을 한 곳에서 관리해야 합니다.
- 운영 전에는 가짜 데이터 제거, 권한 검증, 결제/로그인/푸시 키 연결이 필요합니다.

## 4. 제품 목표

### 1차 목표

1. 고객이 서비스 요청을 만들고 여러 견적을 비교할 수 있게 합니다.
2. 전문가가 견적을 보내고, 채팅하고, 예약을 관리할 수 있게 합니다.
3. 견적 제출 시 전문가 크레딧 40 PHP가 중복 없이 차감되게 합니다.
4. 고객이 완료된 예약에 이용후기를 남길 수 있게 합니다.
5. 관리자가 운영에 필요한 기본 기능을 관리할 수 있게 합니다.
6. Cloudflare 기반 배포 구조로 운영 준비를 마칩니다.

### 성공 기준

- 고객/전문가/관리자 핵심 흐름이 오류 없이 동작합니다.
- 로그인하지 않은 사용자는 보호된 기능에 접근할 수 없습니다.
- 고객은 본인 예약/요청만 볼 수 있고, 전문가는 본인에게 관련된 요청/예약만 볼 수 있습니다.
- 같은 견적 요청에 같은 전문가가 크레딧을 중복 차감당하지 않습니다.
- 완료된 예약에 대해서만 고객이 후기를 한 번 작성할 수 있습니다.
- 운영 키를 연결한 뒤 Google 로그인, PayMongo, Web Push가 실제 환경에서 동작합니다.

## 5. 대상 사용자

### 고객

홈서비스가 필요한 일반 사용자입니다.

주요 행동:

- 서비스 검색
- 견적 요청
- 전문가 비교
- 채팅
- 예약
- 후기 작성

### 전문가

홈서비스를 제공하는 개인 또는 팀입니다.

주요 행동:

- 프로필 작성
- 서비스/기술 등록
- 견적 제출
- 고객과 채팅
- 예약 진행
- 지갑/크레딧 확인

### 관리자

PuntaGo 운영자입니다.

주요 행동:

- 사용자/전문가 상태 관리
- 서비스 카테고리 관리
- 지갑 및 수동 충전 관리
- 추천 보상 설정
- 배너/공지/운영 설정 관리
- 신고/분쟁 관리

## 6. 핵심 사용자 흐름

### 고객 흐름

1. 홈 화면에서 서비스 카테고리 또는 검색어를 선택합니다.
2. 견적 요청서를 작성합니다.
3. 전문가들이 보낸 견적을 확인합니다.
4. 마음에 드는 견적을 선택합니다.
5. 전문가와 채팅합니다.
6. 예약을 진행합니다.
7. 작업 완료 후 이용후기를 작성합니다.

### 전문가 흐름

1. 회원가입 후 전문가 역할을 선택합니다.
2. 전문가 프로필, 기술, 서비스 정보를 등록합니다.
3. 고객의 견적 요청 목록을 확인합니다.
4. 견적을 제출합니다.
5. 견적 제출 시 크레딧 40 PHP가 차감됩니다.
6. 고객과 채팅하고 예약을 진행합니다.
7. 작업 완료 후 평점과 후기를 받습니다.

### 관리자 흐름

1. 관리자 계정으로 로그인합니다.
2. 카테고리, 지갑, 충전, 추천 보상, 운영 설정을 확인합니다.
3. 필요한 데이터를 승인, 수정, 정지, 게시 처리합니다.
4. 운영 전 환경변수와 외부 서비스 연결 상태를 점검합니다.

## 7. 기능 요구사항

### 7-1. 인증/회원

필수 기능:

- 이메일/비밀번호 회원가입
- 로그인/로그아웃
- 역할 선택: 고객, 전문가, 관리자
- Google OAuth 로그인 준비
- 세션 기반 권한 확인

권한 조건:

- 비로그인 사용자는 보호 API 접근 시 401을 받아야 합니다.
- 관리자 API는 관리자만 접근할 수 있어야 합니다.
- 고객/전문가 데이터는 본인 관련 데이터만 접근할 수 있어야 합니다.

### 7-2. 고객 견적 요청

필수 기능:

- 카테고리 선택
- 제목, 설명, 지역, 주소 입력
- 예산 범위 입력
- 희망 날짜 입력
- 내 견적 요청 목록 확인
- 견적 상세 확인

입력 검증:

- 제목과 설명은 비어 있으면 안 됩니다.
- 예산은 숫자여야 합니다.
- 주소와 도시는 필수입니다.

### 7-3. 전문가 견적 제출

필수 기능:

- 받은 견적 요청 목록 확인
- 견적 금액 입력
- 방문 가능일 입력
- 메시지 입력
- 견적 제출

비즈니스 규칙:

- 견적 제출 시 전문가 크레딧 40 PHP를 차감합니다.
- 같은 전문가가 같은 견적 요청에 중복 제출하면 크레딧을 중복 차감하지 않습니다.
- 크레딧이 부족하면 견적 제출을 막아야 합니다.

### 7-4. 예약

필수 기능:

- 견적 선택 후 예약 생성
- 예약 목록 확인
- 예약 상세 확인
- 상태 변경: 대기, 수락, 진행 중, 완료, 취소
- 완료된 예약에만 후기 작성 가능

권한 조건:

- 고객은 본인의 예약만 볼 수 있습니다.
- 전문가는 본인에게 배정된 예약만 볼 수 있습니다.
- 관리자는 운영 목적상 필요한 예약 정보를 확인할 수 있습니다.

### 7-5. 이용후기

필수 기능:

- 완료된 예약의 고객만 후기 작성 가능
- 별점 1~5점
- 후기 내용 10자 이상
- 사진 URL 또는 경로 선택 입력
- 같은 고객은 같은 예약에 후기 1개만 작성 가능
- 후기 작성 후 전문가 평균 평점과 완료 작업 수 갱신

검수 기준:

- 완료되지 않은 예약에는 후기 작성 폼이 보이면 안 됩니다.
- 전문가나 다른 고객이 후기를 작성하면 안 됩니다.
- 중복 작성 시 에러 메시지를 보여야 합니다.

### 7-6. 채팅

필수 기능:

- 견적 요청 또는 예약 기준 대화방 생성
- 메시지 목록 조회
- 텍스트 메시지 전송
- 읽음 표시
- 타이핑 표시
- 파일/이미지 메시지 구조

운영 전 확인:

- 파일 업로드 저장소와 보안 검증은 Cloudflare R2 또는 별도 저장소 기준으로 확정해야 합니다.

### 7-7. 알림

필수 기능:

- 인앱 알림
- 읽음 처리
- 전체 읽음 처리
- 안 읽은 알림 수 표시
- Web Push 구독 저장 구조
- 이메일/SMS/푸시 provider 구조

운영 전 확인:

- Web Push 공개키/비밀키를 실제 운영 값으로 교체해야 합니다.
- 이메일과 SMS 키가 없으면 실제 발송은 동작하지 않습니다.

### 7-8. 지갑/크레딧/결제

필수 기능:

- 전문가 지갑 조회
- 크레딧 충전 패키지: 200 / 400 / 800 PHP
- PayMongo 충전 세션 생성
- PayMongo 웹훅 서명 검증
- 관리자 수동 충전
- 충전 내역 저장
- 견적 제출 수수료 40 PHP 차감

검수 기준:

- 웹훅은 서명이 맞을 때만 처리해야 합니다.
- 같은 결제 이벤트가 중복 처리되면 안 됩니다.
- 수동 충전은 관리자만 가능해야 합니다.

### 7-9. 전문가 상세 페이지

필수 기능:

- 전문가 기본 정보
- 평점과 완료 작업 수
- 인증서
- 작업 사진
- FAQ
- 리뷰
- 서비스 가능 카테고리

### 7-10. 관리자 기능

필수 기능:

- 관리자 대시보드
- 지갑/충전 관리
- 카테고리 관리
- 추천 보상 설정
- 배너/공지 운영 설정
- 긴급 서비스 노출 설정
- 서비스 순위 설정
- 전문가 승인/신고/제재 구조

### 7-11. SEO

필수 기능:

- Open Graph 메타데이터
- Schema.org 구조화 데이터
- sitemap.xml
- robots.txt
- 서비스/전문가 상세 페이지의 검색 노출 준비

## 9. 비기능 요구사항

### 보안

- 서버에서 권한을 반드시 재검사합니다.
- API 입력값은 Zod 스키마로 검증합니다.
- CSRF 보호를 적용합니다.
- 민감 키는 Cloudflare Secrets로 관리합니다.
- 운영 환경에서는 데모 로그인과 데모 데이터를 꺼야 합니다.

### 성능

- Cloudflare Workers 기반으로 전 세계 접속 속도를 높입니다.
- D1 인덱스를 사용해 주요 목록 조회 속도를 유지합니다.
- R2 캐시를 사용해 OpenNext 캐시를 저장합니다.

### 모바일

- 고객 주요 흐름은 모바일에서 먼저 잘 보여야 합니다.
- 버튼과 입력창은 손가락으로 누르기 쉬운 크기여야 합니다.
- 긴 텍스트는 화면 밖으로 넘치지 않아야 합니다.

### 접근성

- 버튼과 입력창에는 의미 있는 라벨을 둡니다.
- 오류 메시지는 사용자가 바로 이해할 수 있어야 합니다.
- 색상만으로 상태를 구분하지 않습니다.

## 10. 운영 전 필수 체크리스트

- Google OAuth 운영 키 입력
- PayMongo 운영 키 입력
- PayMongo Webhook Secret 입력
- Web Push VAPID 키 입력
- Resend/Twilio 키 입력
- Cloudflare D1 원격 마이그레이션 적용 확인
- Cloudflare R2 버킷 연결 확인
- puntago.net 도메인 Worker 연결 확인
- `ENABLE_DEMO_AUTH=false`
- `ENABLE_DEMO_DATA=false`
- lint/test/build 통과
- 고객/전문가/관리자 실제 브라우저 QA 완료

## 11. 현재 위험 요소

- 운영 키가 없으면 Google 로그인, PayMongo 결제, Web Push 실제 발송은 최종 검증할 수 없습니다.
- 도메인 연결이 완성되지 않으면 로그인 후 이동 주소가 깨질 수 있습니다.
- 파일 전송은 DB 구조가 있지만 실제 업로드 저장소 정책을 더 명확히 해야 합니다.
- Cloudflare D1은 일반 PostgreSQL과 다르므로, Prisma/D1 호환 쿼리를 계속 주의해야 합니다.

## 12. 출시 판단 기준

출시 가능:

- 운영 키가 모두 연결됨
- puntago.net 접속 가능
- 실제 Google 로그인 성공
- 실제 PayMongo 결제와 웹훅 성공
- 실제 고객이 견적 요청 생성 가능
- 실제 전문가가 견적 제출 가능
- 크레딧 차감과 중복 방지 확인
- 예약 완료 후 후기 작성 확인

출시 보류:

- 결제 웹훅 실패
- 로그인 리다이렉트 실패
- 권한 없는 사용자가 보호 데이터를 볼 수 있음
- 운영 DB 마이그레이션 미적용
- 데모 데이터가 운영 화면에 노출됨
